# Changelog

## 0.3.1 — 2026-07-02

- Исправлена загрузка служебного JSON вместо видео.
- Перед открытием неизвестной ссылки плагин определяет тип ответа.
- JSON-манифесты разбираются до открытия файла.
- Из полей `src`, `url`, `link` и `file` извлекаются реальные ссылки на видео.
- Добавлен выбор качества для ответов с несколькими вариантами.
- Прямые MP4/WebM/MKV открываются для сохранения на устройство.
- Для HLS после выбора качества предлагаются действия **«Сохранить офлайн»** и **«Смотреть»**.
- Выбранный HLS автоматически передаётся основному загрузчику без повторного выбора качества.
- Стабильный загрузчик переключён на версию 0.3.1.

## 0.3.0 — 2026-07-02

- В первое меню перед запуском видео добавлен пункт **«Скачать на устройство»**.
- Источник текущего видео сохраняется через отдельный listener до перехвата запуска плеером.
- Для прямых MP4/WebM/MKV исходная ссылка открывается в системном просмотрщике без помещения файла в IndexedDB.
- На iPhone файл можно сохранить через «Поделиться» → «Сохранить в Файлы».
- Если автоматическое открытие заблокировано, показывается экран с явной ссылкой **«Открыть видео для сохранения»**.
- Для HLS выводится пояснение, что прямое сохранение одной ссылкой невозможно; остаётся путь через **Сохранить офлайн → В Файлы**.

## 0.2.9 — 2026-07-02

- Исправлена ситуация, когда кнопка открытия меню сохранения на iPhone ничего не делала.
- Web Share API используется только если `navigator.canShare({files})` явно подтверждает поддержку выбранного файла.
- Добавлена настоящая ссылка **«Открыть файл»** на локальный `blob:` URL.
- Добавлена отдельная ссылка **«Скачать»** с атрибутом `download`.
- При ошибке системного меню теперь показывается текст ошибки вместо молчаливого отказа.
- Для больших файлов рекомендуется открыть файл и сохранить его через кнопку «Поделиться» системного просмотрщика iPhone.

## 0.2.8 — 2026-07-02

- Добавлен маркер `Lampa.` для встроенной проверки расширений.
- Исправлен ложный статус `500 / Плагин не подтвержден`.

## 0.2.7 — 2026-07-02

- Добавлена кнопка **«В Файлы»** на обычной и минимальной странице «Офлайн».
- Добавлен список готовых загрузок для экспорта.
- Прямые MP4/WebM/MKV экспортируются через системное меню устройства.
- На iPhone после подготовки файла можно выбрать «Сохранить в Файлы».
- Для браузеров без файлового Web Share API используется fallback через ссылку `download`.
- HLS с `EXT-X-MAP` собирается в `.mp4`, MPEG-TS HLS — в `.ts`.
- Экспорт HLS является best-effort: не каждый поток гарантированно откроется вне Lampa после объединения фрагментов.

## 0.2.6 — 2026-07-02

- Добавлен собственный полноэкранный офлайн-интерфейс без каталога Lampa.
- При `navigator.onLine === false` список читается напрямую из IndexedDB.
- Готовые MP4/WebM/MKV воспроизводятся через локальный `blob:` URL.
- Локальный HLS собирается из сохранённых фрагментов и открывается через Hls.js или нативный HLS.
- Добавлен собственный минимальный `<video>`-плеер с кнопкой закрытия.
- В офлайн-экране можно удалять готовые, недогруженные и ошибочные записи.
- При восстановлении сети минимальный экран скрывается и загружается онлайн-часть 0.2.5.
- Файлы 0.2.6 и 0.2.5 сохраняются в Cache API как best-effort кэш.
- Холодный запуск без загруженного HTML по-прежнему зависит от кэша самого сайта или service worker на домене Lampa.

## 0.2.5 — 2026-07-02

- Карточка выбирается напрямую по `data-id`, без повторного поиска через текущий фокус Lampa.
- Исправлено действие «Смотреть офлайн» на iPhone.
- MP4/WebM/MKV запускаются напрямую из IndexedDB через локальный `blob:` URL.
- Локальный HLS собирается из сохранённых фрагментов и запускается через Hls.js.
- Повторные тапы по списку продолжают работать после закрытия меню или плеера.
- Временные `blob:` URL освобождаются после закрытия плеера.

## 0.2.4 — 2026-07-02

- Добавлена поддержка тапа по карточкам в разделе «Офлайн» на iPhone.
- Тап устанавливает фокус и вызывает то же действие, что `OK / Enter`.
- Добавлена защита от двойного события `touchend + click`.
- Свайп для прокрутки списка не считается нажатием.

## 0.2.3 — 2026-07-02

- Обычный `OK / Enter` теперь открывает меню действий для любой записи.
- Для статусов `queued` и `downloading` добавлено действие «Отменить и удалить».
- Активный сетевой запрос прерывается через `AbortController`.
- Удаляются недогруженные HLS-фрагменты, файл и метаданные из IndexedDB.
- Запись можно удалить из очереди до начала загрузки.
- Долгое нажатие открывает то же меню, но больше не является обязательным.

## 0.2.2 — 2026-07-02

- Сохранённый `blob:...#offline.m3u8` принудительно открывается через Hls.js.
- Исправлена ошибка `The operation is not supported` при локальном HLS-воспроизведении.
- Широкий глобальный CORS-перехват заменён на проверку вызовов только из ядра офлайн-загрузчика.
- Убраны сотни повторяющихся сообщений о переключении credentials.

## 0.2.1 — 2026-07-02

- Исправлена ошибка `Lampa.Controller.active is not a function` в разделе «Офлайн».
- Для медиазапросов загрузчика теперь используется `credentials: omit`.
- Исправлена загрузка MP4 с серверов, которые возвращают `Access-Control-Allow-Origin: *`.
- Добавлена совместимость с текущим API контроллера Lampa.

## 0.2.0 — 2026-07-02

- Добавлен перехват HLS-потоков `.m3u8`.
- Добавлена загрузка master/media-плейлистов и HLS-фрагментов.
- Выбирается вариант с наибольшим `BANDWIDTH`.
- Добавлена поддержка незашифрованного `EXT-X-MAP`.
- Добавлено локальное воспроизведение сохранённого HLS.
- Зашифрованные HLS, DRM и `EXT-X-BYTERANGE` явно отклоняются.
- Создан отдельный установочный файл, чтобы исключить кэш версии 0.1.0.

## 0.1.0 — 2026-07-02

- Перехват запуска прямого видео через `Lampa.Player.listener`.
- Выбор: смотреть, скачать, смотреть и скачать.
- Последовательная очередь загрузок.
- Хранение в OPFS с fallback на IndexedDB.
- Раздел «Офлайн» в главном меню.
- Запуск, отмена, повтор и удаление файлов.
- Настройки запроса перед воспроизведением и очистки хранилища.
