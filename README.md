# Lampa Offline MVP

Плагин добавляет в Lampa сохранение прямых видеофайлов и незашифрованных HLS-потоков для просмотра без интернета.

> Используйте плагин только для контента, который разрешено сохранять. Он не обходит DRM, шифрование и защиту видеосервисов.

## Установка версии 0.2.1

Версия 0.2.1 исправляет совместимость с актуальной Lampa и загрузку прямых MP4 из `modss`.

Если установлена версия 0.2.0 или более старая, сначала удалите её из списка расширений Lampa. Одновременно оставлять несколько версий нельзя: они будут перехватывать один и тот же запуск плеера.

1. Откройте **Lampa → Настройки → Расширения**.
2. Удалите старый адрес офлайн-плагина.
3. Нажмите **Добавить плагин**.
4. Вставьте адрес:

```text
https://cdn.jsdelivr.net/gh/iteaky/lampa@main/lampa-offline-mvp-0.2.1.js
```

5. Подтвердите добавление.
6. Полностью закройте Lampa, выгрузите её из памяти и откройте снова.

Используется новое имя файла, чтобы устройство не получило из кэша старую версию.

## Как пользоваться

1. Откройте фильм или сериал через установленный онлайн-плагин, например `modss`.
2. Выберите перевод, качество или серию и запустите видео.
3. Перед открытием плеера должно появиться меню:
   - **Смотреть**;
   - **Сохранить офлайн**;
   - **Смотреть и сохранить**.
4. Выберите **Сохранить офлайн**.
5. Откройте раздел **Офлайн** в главном меню Lampa, чтобы увидеть загрузку.
6. На готовой записи нажмите **OK / Enter**, чтобы посмотреть локальную копию.
7. Долгое нажатие **OK / Enter** удаляет запись.

Если загрузка завершилась ошибкой, обычное нажатие на неё запускает повторную попытку.

## Что исправлено в 0.2.1

- исправлена ошибка `Lampa.Controller.active is not a function` в разделе **Офлайн**;
- для медиазапросов загрузчика используется `credentials: omit`, поэтому серверы с `Access-Control-Allow-Origin: *` больше не конфликтуют с cookies;
- исправление применяется только к GET/CORS-запросам с сигнатурой офлайн-загрузчика и не затрагивает авторизацию `modss`;
- стабильный файл `lampa-offline-mvp.js` теперь загружает версию 0.2.1.

## Что умеет версия 0.2.x

- перехватывает прямые MP4/WebM/MKV-ссылки;
- перехватывает HLS-ссылки `.m3u8`;
- выбирает вариант с наибольшим `BANDWIDTH` из master-плейлиста;
- загружает HLS-плейлист и все его видеофрагменты;
- поддерживает `EXT-X-MAP` для незашифрованного fragmented MP4;
- собирает локальный плейлист для воспроизведения без интернета;
- показывает очередь, прогресс, повтор и удаление загрузок;
- восстанавливает незавершённую очередь после перезапуска.

## Ограничения

Не поддерживаются:

- DRM/Widevine;
- HLS с `EXT-X-KEY` и любым шифрованием;
- HLS с `EXT-X-BYTERANGE`;
- MPEG-DASH (`.mpd`);
- отдельная внешняя аудиодорожка из сложного master-плейлиста;
- источники, требующие специальных HTTP-заголовков;
- загрузка при полностью закрытой Lampa;
- продолжение с точного байта после закрытия приложения.

Сервер должен разрешать JavaScript-запросы из Lampa. Даже после исправления `credentials` отдельные видеосерверы могут полностью запрещать CORS-загрузку.

## Если меню или загрузка не работают

- убедитесь, что старая версия удалена;
- проверьте, что установлен именно адрес с `0.2.1`;
- полностью выгрузите Lampa из памяти и запустите снова;
- проверьте наличие раздела **Офлайн** в главном меню;
- временно удалите другие плагины, которые тоже изменяют запуск плеера;
- откройте консоль и найдите строки `[Offline HLS compat]` и `[Offline HLS]`.

## Файлы

- `CHANGELOG.md` — история изменений.

## Проверка синтаксиса

```bash
node --check lampa-offline-mvp-0.2.1.js
node --check lampa-offline-mvp-0.2.0.js
```

Полноценная проверка загрузки выполняется на устройстве внутри Lampa, поскольку результат зависит от WebView, CORS и формата конкретного потока.
