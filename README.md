# Lampa Offline MVP

Плагин добавляет в Lampa сохранение прямых видеофайлов для просмотра без интернета.

> MVP предназначен только для контента, который разрешено сохранять. Плагин не обходит DRM и защиту видеосервисов.

## Быстрая установка

1. Откройте **Lampa**.
2. Перейдите в **Настройки → Расширения → Добавить плагин**.
3. Вставьте адрес:

```text
https://cdn.jsdelivr.net/gh/iteaky/lampa@main/lampa-offline-mvp.js
```

4. Подтвердите добавление плагина.
5. Полностью перезапустите Lampa.
6. После запуска в меню должен появиться раздел **Офлайн**, а перед воспроизведением подходящего видео — выбор действия.

## Альтернативный адрес через GitHub Pages

После включения GitHub Pages для ветки `main` и каталога `/root` можно использовать:

```text
https://iteaky.github.io/lampa/lampa-offline-mvp.js
```

Включение Pages: **GitHub → Settings → Pages → Deploy from a branch → main → /root → Save**.

## Как пользоваться

1. Откройте фильм или сериал через обычный онлайн-источник Lampa.
2. Выберите конкретное видео или серию.
3. Для поддерживаемой прямой ссылки появится меню:
   - **Смотреть**;
   - **Сохранить офлайн**;
   - **Смотреть и сохранить**.
4. Ход загрузки отображается в разделе **Офлайн**.
5. Нажмите Enter/OK на готовом элементе, чтобы открыть локальную копию.
6. Удерживайте Enter/OK, чтобы отменить, повторить или удалить загрузку.

## Что умеет версия 0.1.0

- сохраняет прямые MP4, WebM и MKV-ссылки;
- показывает очередь и прогресс загрузки;
- загружает файлы последовательно;
- хранит большие файлы в OPFS, а при его отсутствии использует IndexedDB;
- восстанавливает незавершённые элементы очереди после перезапуска;
- запускает сохранённое видео через плеер Lampa;
- позволяет очистить офлайн-хранилище в **Настройки → Офлайн**.

## Ограничения MVP

Пока не поддерживаются:

- HLS (`.m3u8`) и MPEG-DASH (`.mpd`);
- DRM/Widevine и обход любой защиты;
- торрент-потоки;
- временные `blob:`-ссылки;
- источники, которым нужны нестандартные HTTP-заголовки;
- загрузка после принудительного закрытия Lampa операционной системой;
- продолжение загрузки с конкретного байта после полного закрытия приложения.

Видеосервер должен разрешать запросы из Lampa. Из-за CORS возможна ситуация, когда видео воспроизводится, но загрузка через JavaScript блокируется сервером.

## Если плагин не появился

- убедитесь, что адрес добавлен целиком и начинается с `https://`;
- полностью закройте и снова откройте Lampa;
- удалите старую запись плагина и добавьте её заново;
- проверьте, что используете актуальную версию Lampa;
- попробуйте альтернативный адрес через GitHub Pages;
- учтите, что офлайн-хранилище зависит от браузера/WebView и свободного места на устройстве.

## Разработка

Основной файл:

```text
lampa-offline-mvp.js
```

Проверка синтаксиса:

```bash
node --check lampa-offline-mvp.js
```

Реальную загрузку нужно тестировать внутри Lampa, поскольку она зависит от CORS, WebView и доступного хранилища.
