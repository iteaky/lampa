# Lampa Offline MVP

Плагин добавляет в Lampa сохранение прямых видеофайлов и незашифрованных HLS-потоков для просмотра без интернета.

> Используйте плагин только для контента, который разрешено сохранять. Он не обходит DRM, шифрование и защиту видеосервисов.

## Установка версии 0.2.3

Версия 0.2.3 исправляет управление недогруженными записями: теперь обычное нажатие **OK / Enter** открывает меню действий для любого состояния загрузки.

Если установлена предыдущая версия, сначала удалите её из списка расширений Lampa. Одновременно оставлять несколько версий нельзя.

1. Откройте **Lampa → Настройки → Расширения**.
2. Удалите старый адрес офлайн-плагина.
3. Нажмите **Добавить плагин**.
4. Вставьте адрес:

```text
https://cdn.jsdelivr.net/gh/iteaky/lampa@main/lampa-offline-mvp-0.2.3.js
```

5. Подтвердите добавление.
6. Полностью закройте Lampa, выгрузите её из памяти и откройте снова.

## Как пользоваться

1. Откройте фильм или сериал через онлайн-плагин.
2. Выберите перевод, качество или серию и запустите видео.
3. Перед открытием плеера появится меню:
   - **Смотреть**;
   - **Сохранить офлайн**;
   - **Смотреть и сохранить**.
4. Прогресс отображается в разделе **Офлайн**.

## Действия с загрузкой

Выделите любую запись в разделе **Офлайн** и нажмите обычный **OK / Enter**.

Для готовой записи доступны:

- **Смотреть офлайн**;
- **Удалить**.

Для загрузки со статусом **В очереди** или **Загрузка** доступны:

- **Отменить и удалить**;
- **Удалить**.

Для записи со статусом **Ошибка** или **Отменено** доступны:

- **Повторить загрузку**;
- **Удалить**.

Долгое нажатие больше не обязательно: оно открывает то же меню.

## Что исправлено в 0.2.3

- обычный OK открывает меню для всех состояний;
- добавлено действие **Отменить и удалить** для активной загрузки;
- текущий сетевой запрос прерывается через `AbortController`;
- недогруженные HLS-фрагменты и метаданные удаляются из IndexedDB;
- запись в очереди можно удалить до начала загрузки;
- сохранены исправления локального HLS, CORS и совместимости контроллера из версий 0.2.1–0.2.2;
- стабильный файл `lampa-offline-mvp.js` переключён на 0.2.3.

## Возможности версии 0.2.x

- прямые MP4, WebM и MKV;
- незашифрованные HLS-потоки `.m3u8`;
- выбор варианта с наибольшим `BANDWIDTH`;
- загрузка HLS-плейлиста и видеофрагментов;
- поддержка незашифрованного `EXT-X-MAP`;
- локальное воспроизведение через Hls.js;
- очередь, прогресс, повтор и удаление загрузок.

## Ограничения

Не поддерживаются:

- DRM/Widevine;
- HLS с `EXT-X-KEY` и шифрованием;
- HLS с `EXT-X-BYTERANGE`;
- MPEG-DASH (`.mpd`);
- отдельная внешняя аудиодорожка из сложного master-плейлиста;
- источники, требующие специальных HTTP-заголовков;
- загрузка при полностью закрытой Lampa;
- продолжение с точного байта после закрытия приложения.

Ответ `423 Locked` приходит от самого видеосервера. Такую ссылку нельзя скачать до получения новой рабочей ссылки от онлайн-плагина.

## Диагностика

После запуска в консоли должны появиться строки:

```text
[Offline HLS compat] Controller.active compatibility installed
[Offline HLS compat] scoped fetch compatibility installed
[Offline HLS compat] offline HLS playback compatibility installed
[Offline HLS] initialized 0.2.0
[Offline HLS compat] initialized 0.2.3
```

При открытии раздела **Офлайн** должна появиться строка:

```text
[Offline HLS compat] offline item actions installed
```

## Файлы

- `lampa-offline-mvp-0.2.3.js` — актуальный совместимый загрузчик;
- `lampa-offline-mvp-0.2.0.js` — основное ядро MP4/HLS;
- `lampa-offline-mvp.js` — стабильный загрузчик актуальной версии;
- `CHANGELOG.md` — история изменений.

## Проверка синтаксиса

```bash
node --check lampa-offline-mvp-0.2.3.js
node --check lampa-offline-mvp-0.2.0.js
```

Полноценная проверка выполняется внутри Lampa, поскольку результат зависит от WebView, CORS и формата потока.
